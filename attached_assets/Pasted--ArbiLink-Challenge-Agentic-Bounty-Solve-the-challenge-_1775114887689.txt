## What We’re Looking For

---

Build a **skill, framework extension,** or a **full agentic workflow** that enables an AI agent to interact with Arbitrum. This includes things like:

- Deploying smart contracts on Arbitrum
- Integrating wallets (EOA, smart wallets, session keys)
- Bridging assets across chains
- Reading on-chain data (prices, balances, events)
- Interacting with Arbitrum ecosystem protocols (Uniswap, GMX, Camelot, Fhenix, etc.)
- Running or connecting to Arbitrum nodes

**Bonus points for:**

- ⭐ Original skills not seen before in the ecosystem
- ⭐ Skills that integrate a specific Arbitrum ecosystem project

### **Step 2: Register Your Agent (Mandatory)**

Register your agent on the Arbitrum identity registry using the official agent0 SDK, the Arbitrum-specific implementation, or your own approach.

Arbitrum One registry: [`0x8004A169FB4a3325136EB29fA0ceB6D2e539a432`](https://arbiscan.io/address/0x8004A169FB4a3325136EB29fA0ceB6D2e539a432)
Arbitrum Sepolia registry: [`0x8004A818BFB912233c491871b3d84c89A494BD9e`](https://sepolia.arbiscan.io/address/0x8004A818BFB912233c491871b3d84c89A494BD9e) 

### **Step 3: Deploy & Go Agentic (Bonus)**

Take your submission further by deploying a live, interactive agent, and this is where you can get creative with the stack.

This can be:

- A deployed agent with a public HTTP endpoint
- A **Pi.dev** agent extension
- An **OpenClaw** plugin or workflow
- Any other agent framework integration that runs on or connects to Arbitrum

Your implementation should be publicly accessible and include a link to your own repository with documentation.

This step is **optional but recommended;** submissions with a live implementation will get additional points for the top 3 prizes.
